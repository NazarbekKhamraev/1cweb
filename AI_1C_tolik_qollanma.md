# AI билан 1С дастурлаш: А дан Я гача тўлиқ қўлланма

Бу қўлланма орқали сиз ноллдан бошлаб қуйидаги тизимни ишга тушира оласиз:

- **Claude Code** — терминалда ишлайдиган AI-дастурчи ёрдамчи;
- **cc-1c-skills** — Claude'га 1С да тўғри код ёзишни ўргатадиган навыклар тўплами;
- **BSL Language Server** — AI ёзган кодни автоматик, қоида бўйича текширадиган восита.

**Натижа:** сиз масалани оддий сўз билан айтасиз → Claude 1С обработка/ҳисобот/форма яратади → BSL Language Server уни текширади → хатолар тузатилади. Ҳаммаси битта муҳитда.

⏱ **Керакли вақт:** биринчи марта тахминан 30–40 дақиқа.

---

## 0. Бошлашдан олдин: керакли нарсалар

Қуйидагилар тайёр бўлсин:

- ✅ **Windows 10 ёки 11**
- ✅ **1С:Предприятие 8.3** ўрнатилган (обработка/ҳисоботларни йиғиш учун)
- ✅ Барқарор интернет
- ✅ **Claude Pro / Max обунаси** ёки **API калит** (Claude Code учун)
- ✅ Тахминан 1 соат бўш вақт

Биз қуйидаги тартибда кетамиз:
**Node.js → Java → VS Code → Claude Code → cc-1c-skills → BSL Language Server → уланиш → биринчи тест.**

> Ҳар қадамни кетма-кет бажаринг, ошиб ўтманг. Ҳар қадам охирида "текшириш" бор — ишлаганига ишонч ҳосил қилиб, кейингисига ўтинг.

---

## 1-қадам. Node.js ўрнатиш

**Нега керак:** Claude Code ва cc-1c-skills'нинг веб-тест қисми Node.js'да ишлайди.

1. [nodejs.org](https://nodejs.org) га киринг → **LTS** версияни (v20 ёки v22) юклаб олинг.
2. Ўрнатувчини ишга туширинг. "Add to PATH" белгиси турганига эътибор беринг.
3. Ўрнатиб бўлгач, **терминални (ёки VS Code'ни) қайта очинг**.

**Текшириш:**
```bash
node --version
npm --version
```
`node` версияси **v18 дан юқори** бўлса — тайёр.

---

## 2-қадам. Java ўрнатиш

**Нега керак:** BSL Language Server — Java дастури, унинг ишлаши учун Java керак.

1. [adoptium.net](https://adoptium.net) га киринг → **OpenJDK 21 (Temurin)** ни Windows учун юклаб олинг.
2. Ўрнатинг (келишув бўйича — "Add to PATH" ёқилган ҳолда).

**Текшириш:**
```bash
java -version
```
Версия **17, 21 ёки 23** бўлса — тайёр.

---

## 3-қадам. VS Code ўрнатиш

**Нега керак:** бу — код ёзиш, кўриш ва хатоларни кузатиш муҳити.

1. [code.visualstudio.com](https://code.visualstudio.com) → **Download** → ўрнатинг.
2. VS Code'ни очинг. Ичидаги терминал: **Terminal → New Terminal** (`Ctrl+~`).

---

## 4-қадам. Claude Code ўрнатиш ва кириш

**Нега керак:** бу — асосий AI-дастурчи ёрдамчингиз.

1. Терминалда:
```bash
npm install -g @anthropic-ai/claude-code
```
> ⚠️ Хеч қачон `sudo` билан ўрнатманг — рухсат хатоларига олиб келади.

2. **Текшириш:**
```bash
claude --version
```

3. Лойиҳа папкангизга кириб, `claude` деб ишга туширинг. Биринчи марта браузер очилиб, Anthropic аккаунтингизга кириш сўрайди (**Pro/Max обуна** ёки **API калит** керак).

4. Муаммо бўлса, диагностика:
```bash
claude doctor
```

> 💡 **Илғорлар учун:** Claude Code'ни бошқа модель провайдерларига ҳам (environment variable орқали) йўналтириш мумкин — бу алоҳида мавзу, ҳозир стандарт йўлдан кетамиз.

## 5-қадам. Лойиҳа папкасини тайёрлаш

1. Битта бўш папка яратинг, масалан: `C:\1c-ai\MyProject`
2. VS Code'да шу папкани очинг (**File → Open Folder**).
3. Ичидаги терминални очинг — кейинги буйруқларни шу ерда терасиз.

## 6-қадам. cc-1c-skills ўрнатиш

**Нега керак:** бу навыклар Claude'га 1С'нинг XML форматлари ва конфигуратор буйруқларини "ўргатади", шунда у тўғри код/файл яратади.

**Талаблар:** Windows + PowerShell, 1С 8.3 (йиғиш учун), Node.js (веб-тест учун).

Икки йўлдан бирини танланг:

**А) Плагин орқали (осон)** — Claude Code ичида:
```
/plugin marketplace add https://github.com/Nikolay-Shirokov/cc-1c-skills
/plugin install 1c-skills@cc-1c-skills
```

**Б) Қўлда** — репозиторийдан `.claude/skills/` папкасини лойиҳа илдизига нусхаланг:
```
MyProject/
├── .claude/skills/    ← шу ерга нусхаланг
└── ...
```

**Текшириш:** Claude'га оддий тил билан ёзинг:
```
Создай обработку ТестОбработка с формой
```
Агар Claude керакли навыкни ўзи танлаб, тўғри структурани яратса — ишлаяпти. (Буйруқларни ёдлаш шарт эмас — оддий сўз билан айтсангиз бўлди.)

## 7-қадам. BSL Language Server — VS Code тарафи

**Нега керак:** код ёзаётганда хатоларни муҳарририда дарҳол кўрасиз.

1. VS Code → **Extensions** (`Ctrl+Shift+X`) → қидирувга **`Language 1C (BSL)`** (муаллиф — **1c-syntax**) → **Install**.
2. Бирор `.bsl` файлни очинг → хатолар **Problems** панелида (`Ctrl+Shift+M`) кўринади. Метод мураккаблиги ҳам кўрсатилади.
3. (ихтиёрий) Тил созлаш — лойиҳа илдизида `.bsl-language-server.json`:
```json
{ "language": "ru" }
```

---

## 8-қадам. BSL Language Server `.jar` файлини юклаб олиш

**Нега керак:** Claude Code'га улаш учун сервернинг мустақил `.jar` файли керак.

1. [github.com/1c-syntax/bsl-language-server/releases](https://github.com/1c-syntax/bsl-language-server/releases) → энг сўнгги версиядан ишга тушадиган **`.jar`** файлни (одатда `...-exec.jar`) юклаб олинг.
2. Қулай жойга қўйинг: `C:\tools\bsl-language-server.jar`

**Текшириш:**
```bash
java -jar C:/tools/bsl-language-server.jar --version
```

---

## 9-қадам. BSL Language Server'ни Claude Code'га улаш (MCP)

**Нега керак:** шунда Claude ёзган кодни воситанинг ўзи ўша заҳоти текширади.

Лойиҳа папкасидаги терминалда:
```bash
claude mcp add bsl-ls -- java -jar C:/tools/bsl-language-server.jar mcp
```

Ёки лойиҳа илдизида `.mcp.json` файл яратинг:
```json
{
  "mcpServers": {
    "bsl-language-server": {
      "command": "java",
      "args": ["-jar", "C:/tools/bsl-language-server.jar", "mcp"]
    }
  }
}
```

**Текшириш:** Claude Code ичида `/mcp` — server ва tool'лар рўйхатини кўрсатади.

> ⚠️ MCP режими ҳали **эксперименталь**. Ишламай қолса — 10-қадамдаги CLI усули ҳам бор, у доим ишлайди.

## 10-қадам. CLAUDE.md — AI ўзини текширадиган қоида

**Нега керак:** ҳар сафар қўлда айтмаслик учун. Claude код ёзгач ўзи текширади ва тузатади.

Лойиҳа илдизида `CLAUDE.md` файл яратинг ва ичига ёзинг:

```markdown
## BSL кодни текшириш қоидаси

Ҳар .bsl ёки .os файл яратгач ёки ўзгартиргач:

1. Ишга тушир:
   java -jar C:/tools/bsl-language-server.jar --analyze --srcDir ./src --reporter json --outputDir ./bsl-report
2. ./bsl-report ичидаги JSON ҳисоботни ўқи.
3. "Error" даражасидаги барча хатоларни туза.
4. Тоза бўлгунча 1–3 қадамларни такрорла.

(Агар MCP уланган бўлса: буйруқ ўрнига analyze_file tool'ини ишлат.)
```

## 11-қадам. Биринчи амалий вазифа (тизимни синаб кўрамиз)

Claude Code ичида ёзинг:
```
Создай внешнюю обработку ТестОбработка с управляемой формой.
```

## Муаммоларни ҳал қилиш (Troubleshooting)

- **`claude` топилмади** → терминални тўлиқ ёпиб қайта очинг; PATH'ни текширинг.
- **`java` топилмади** → Java ўрнатилганини ва PATH'да борлигини текширинг.
- **Сборка (йиғиш) ишламаяпти** → 1С:Предприятие 8.3 ўрнатилганини текширинг (XML яратиш 1С'сиз ҳам ишлайди, лекин `.epf` йиғиш учун 1С керак).
- **MCP уланмаяпти** → 10-қадамдаги CLI усулига ўтинг, у доим ишлайди.
- **Йўллар хатоси** → Windows'да `.mcp.json` ва буйруқларда `/` ишлатинг: `C:/tools/...`
- **Катта лойиҳа секин** → хотирани оширинг: `java -Xmx4g -jar ...`
